Project Description

Game engine is very helpful thing for game programmers. A simple game engine is just a library with game programming related classes or functions. How I become game engine developer? OK, I explain it.

I bought RZX-50 because I like emulation and old-school video games. RZX-50 similar to Dingoo A320, but RZX-50 use Dingux as firmware by default. Dingux is Linux-based operating system. I made my own multimedia library for Dingux, because I want test myself as Linux programmer. This library has a cross-platform layered architecture. It has a two kinds of subsystems: system-specific and common subsystems. Common subsystems work over system-specif subsystems.

I also made few multimedia libraries for Microsoft Windows. I did it by easy way: just forked my Dingux library and replace all system-specific subsystems.

Features

A good game engine should have optimal set of features. Let's look to features of my game engines.

Common features:

  • Easy to use object-orientated API.
  • Graphics primitives.
  • Animated and static sprites.
  • Scrollable and static backgrounds.
  • Tile-maps.
  • Image transformation subsystem: resizing and mirroring.
  • Image loader with PCX and TGA support.
  • Text drawing subsystem.
  • Custom fonts.
  • Timers.
  • Collision detector.
  • Integrated fps counter.

Microsoft Windows specific features:

  • Powerful input subsystem: keyboard, mouse, game-pad.
  • Simple multimedia subsystem for playing audio and video.
  • Coordinate transformation subsystem.

Dingoo Linux specific features:

  • Simple input subsystem.
  • Fast render via frame-buffer.
