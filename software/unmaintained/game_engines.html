Project Description

Game engine is very helpful thing for game programmers. A simple game engine is just a library with game programming related classes or functions. How I become game engine developer? OK, I explain it.

I like emulation and old-school video games. I also like emulation-orientated portable game consoles. Some of these consoles use Embedded Linux as firmware. I made my own multimedia library for some of these consoles, because I want test myself as Linux programmer. This library has a cross-platform layered architecture. It has a two kinds of subsystems: system-specific and common subsystems. Common subsystems work over system-specif subsystems.

I also made few multimedia libraries for Microsoft Windows. I did it by easy way: just forked my Linux library and replace all system-specific subsystems.

Features

A good game engine should have optimal set of features. Let's look to features of my game engines.

Common features:

  • Easy to use object-orientated API.
  • Graphics primitives.
  • Animated and static sprites.
  • Scrollable and static backgrounds.
  • Tile-maps.
  • Image transformation subsystem: resizing and mirroring.
  • Image loader with PCX and TGA support.
  • Text drawing subsystem.
  • Custom fonts.
  • Timers.
  • Collision detector.
  • Integrated fps counter.

Microsoft Windows specific features:

  • Powerful input subsystem: keyboard, mouse, game-pad.
  • Simple multimedia subsystem for playing audio and video.
  • Coordinate transformation subsystem.

Embedded Linux specific features:

  • Simple input subsystem.
  • Display back-light control subsystem.
  • Fast render via frame-buffer.
  • Low-level sound subsystem.
  • Simple sound mixer subsystem.
  • Audio loader with Waveform Audio support.
  • Simple audio playing subsystem.
